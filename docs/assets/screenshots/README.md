# Screenshot Capture Checklist

Add real screenshots here before a public release. Capture only synthetic or
sample data. Do not show real health information, real medication schedules,
symptoms, Firebase tokens, service account keys, user exports, raw audit logs,
real emails, full UIDs, or credential paths.

## Required Files

- `dashboard.png`: app home or dashboard view that shows the prototype identity.
- `meal-entry.png`: synthetic meal entry flow.
- `conflict-result.png`: synthetic conflict-analysis result with educational
  explanation and safety wording visible.

## Capture Requirements

- Use local mode or a safe synthetic-data demo account.
- Keep text readable at GitHub README size.
- Prefer desktop browser screenshots for README display.
- Crop browser chrome only if it does not hide important context.
- Review screenshots against `DISCLAIMER.md` before committing.

## README Linking Policy

Only embed image files in `README.md` after the real screenshots exist in this
directory. Until then, link to this checklist instead of publishing broken image
links or mocked clinical screenshots.
