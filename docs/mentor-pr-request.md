# Mentor/Classmate Starter PR Request

ParkinSUM should show real external participation through small, honest
contributions. Do not create fake forks, watchers, stars, or pull requests.

## Suggested starter PRs

Choose one narrow task:

- Improve one README sentence for clarity without changing the safety boundary.
- Test the Quick Start commands on a clean machine and update wording if a step
  is confusing.
- Add or improve alt text for one synthetic screenshot.
- Review the GitHub Pages landing page for teacher, mentor, or digital-health
  audience clarity.

## Contributor checklist

- Use synthetic or sample data only.
- Do not include personal health information, real medication schedules,
  screenshots of private accounts, credentials, or local machine paths.
- Do not claim diagnosis, treatment, clinical validation, patient outcome
  improvement, or medical-device status.
- Keep the PR small enough to review in one pass.

## Copy-ready request

Hi, I am preparing ParkinSUM as a public educational open-source prototype. Could
you make one small PR as an external reviewer? Good options are README wording,
Quick Start verification, screenshot alt text, or a small GitHub Pages wording
improvement. Please use synthetic/demo examples only and avoid any medical
advice or clinical-validation claims.
